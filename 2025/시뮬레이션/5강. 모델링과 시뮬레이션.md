### 시뮬레이션 모델의 이용 범주

- 설명적 장치: 시스템이나 문제를 정의
- 분석 도구: 한계적 구성요소 결정
- 설계평가도구: 제안된 해결방안을 종합하고 평가
- 예측 도구: 미래의 개발계획을 예측



### 시스템(system)

- 어떤 목적을 위해 하나 이상 서로 관련된 구성요소들이 결합된 집합 → 모델(model) (시스템을 서술한 것)
  - 시스템의 범위 결정 문제
  - 외적 요인들
    - 포함하거나
    - 무시하거나
    - 입력변수로 함



### 모델 설계

- **용이한 경우**
  - **물리적 규칙** 이용 가능
  - **도형적 표현**이 가능
  - **입력, 출력, 구성요소의 변화가 통제 가능**
- 어려운 경우
  - 물리적 기본 규칙 없음
  - 표현하기 어려운 많은 절차적 요소
  - 랜덤 구성 요소
  - 정량화가 어려운 정책적
  - 인간의 의사결정이 큰 영향을 주는 경우

## 2. 시뮬레이션 모델의 종류 및 예

### 시뮬레이션 모델의 종류

- 결정적 모델과 확률적 모델
  - 결정적 모델: 저축문제 1, 공의 탄성 문제
  - 확률적 모델: 주사위문제, 저축문제 2
- 정적 모델과 동적 모델
  - 정적 모델: 시간과 무관, 몬테칼로 시뮬
  - 동적 모델: 시간의 흐름에 따라 동적으로 변함
- 이산 모델과 연속 모델
  - 이산 모델: 대기행렬문제
  - 연속적 모델: 공의탄성문제
- 물리적 모델과 수리적 모델
  - 물리적 모델: 저축문제, 공의 탄성문제
  - 수리적 모델: 진흙모델, 모델하우스, 축소모형

## 4. 시뮬레이션 특징과 과정

### 시뮬레이션 특징(장점)

- 실제 구축 없이 예측 평가 가능
- 해석적 방법이 불가능한 경우 실험적 대안
  - 실시스템은 매우 복잡해 공식화 하기 어려움
- 비용이 높고, 위험한 경우
- 다양한 환경 설정과 반복실험 가능
- 장(경제시스템), 단기간 시스템 연구 가능



### 시뮬레이션 특징(단점)

- 개발에 많은 비용과 시간 소요
- 프로그래밍 연습이라는 인상 줄 수 있음
- 과잉 신뢰 주는 경향 있음
- 확률적 시뮬레이션은 추정량을 산출할 뿐
  - 비교 평가에는 적합, 최적화에는 부적합
